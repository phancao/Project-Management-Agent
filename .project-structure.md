# Project Structure

This document describes the organization of the Project Management Agent project.

## Root Directory

### Core Application Files
- `main.py` - Main application entry point
- `server.py` - Server entry point
- `run_tests.py` - Test runner
- `quick_test.py` - Quick test script
- `bootstrap.sh` / `bootstrap.bat` - Bootstrap scripts

### Configuration Files
- `conf.yaml` - Runtime configuration (not in version control)
- `pyproject.toml` - Python project configuration
- `uv.lock` - Dependency lock file
- `langgraph.json` - LangGraph configuration
- `docker-compose.yml` - Docker Compose configuration
- `Dockerfile` / `Dockerfile.api` - Docker build files
- `.env` - Environment variables (not in version control)

### Documentation
- `README.md` - Main project documentation
- `README_*.md` - Translated READMEs
- `CONTRIBUTING` - Contribution guidelines
- `LICENSE` - License file

## Directory Structure

### `backend/` - Source Code
Main application source code organized by feature/module.

### `scripts/` - Utility Scripts
- `utils/` - Utility scripts (provider management, database, setup)
- `fixes/` - Fix/repair scripts
- `dev/` - Development helper scripts
- `tests/` - Standalone test scripts

### `docs/` - Documentation
- `guides/` - Comprehensive guides
- `reports/` - Implementation summaries and reports
- Main documentation files

### `tests/` - Official Test Suite
Automated unit and integration tests (pytest).

### `config/` - Configuration Templates
Configuration file templates and examples.

### `logs/` - Log Files
Application log files (not in version control).

### `web/` - Frontend Application
Next.js frontend application.

### `database/` - Database
Database schemas, migrations, and utilities.

### `api/` - API Module
API-specific code.

### `assets/` - Assets
Static assets and resources.

### `examples/` - Examples
Example files and demonstrations.
